# 实验样本接收登记系统

本地桌面工具，用于实验样本的接收登记、状态流转、数据导出。不依赖任何外部 LIMS 系统，所有数据本地存储。

## 功能特性

### 核心流程
- **样本录入**：支持手动单条录入或批量导入（CSV/JSON）
- **数据校验**：样本编号、项目、数量、接收人、存放位置
- **状态流转**：已接收 → 待补资料 / 已入库 / 已退回 / 已作废
- **操作拒绝**：重复样本号、已退回样本直接入库、导出目录异常等都会被明确拒绝。已退回样本入库有双重防护：GUI层按钮禁用 + 数据库层校验拒绝，失败操作不修改任何数据（状态、历史、备注保持不变），无二次确认、无异常标记

### 失败路径覆盖
1. **重复样本号**：导入或录入时自动检测重复编号并拒绝
2. **退回样本直接入库**：操作被明确拒绝，状态、历史、备注保持不变，必须先"重新接收"再入库
3. **导出目录不存在/不可写**：导出前校验路径，失败不覆盖原记录

### 持久化能力
- SQLite 本地数据库存储
- 关闭重开后仍可查看：
  - 所有样本状态历史
  - 破损/缺管备注
  - 上次导出配置（目录、格式、筛选条件）

### 导出功能
- 支持按日期范围、项目、状态筛选
- 导出格式：CSV / JSON
- 可选择包含状态历史

### 界面功能
- 待处理样本筛选（已接收、待补资料）
- 单条样本时间线查看
- 破损/缺管备注编辑
- 操作员配置持久化

## 样例数据操作步骤

### 步骤 1：设置操作员
1. 启动程序后，在顶部"操作员"输入框填写姓名，例如 `测试员`
2. 点击"保存操作员"

### 步骤 2：导入样例数据
1. 点击工具栏的"📂 导入 CSV"按钮
2. 选择项目目录下的 `sample_data.csv` 文件
3. 弹出"导入选项"对话框，选择"是"（遇到错误时跳过继续）
4. 查看导入结果提示：成功 10 条
5. 再点击"📂 导入 JSON"按钮
6. 选择 `sample_data.json` 文件，同样选择"是"跳过错误
7. 查看导入结果：成功 5 条

> **注意**：如果重复导入同一份文件，系统会检测到重复样本号并报错，已成功导入的记录不会被覆盖。

### 步骤 3：手动录入样本
1. 点击"📝 手动录入"按钮
2. 填写样本信息：
   - 样本编号：`TEST-999`
   - 项目：`新冠检测`
   - 数量：`3`
   - 接收人：`测试员`
   - 存放位置：`冰箱Z-第9层`
   - 破损备注：（可选）
   - 缺管备注：（可选）
3. 点击"确认登记"
4. 尝试再次录入相同编号，系统会提示"样本编号已存在"

### 步骤 4：筛选待处理样本
1. 在"筛选条件"区域，状态选择"待处理"
2. 点击"🔍 查询"
3. 列表显示状态为"已接收"和"待补资料"的样本

### 步骤 5：状态流转
1. 选中一条"已接收"状态的样本（如 TEST-001）
2. 点击底部"→ 已入库"按钮
3. 输入备注：`资料齐全，入库`，点击确定
4. 观察列表中该样本状态变为"已入库"，背景色变为蓝色

### 步骤 6：退回样本重新入库流程
1. 选中另一条"已接收"样本
2. 点击"→ 已退回"，备注：`资料不全退回`
3. 选中这条"已退回"样本（紫色背景）
4. 观察到"→ 已入库"按钮已禁用，"← 重新接收"按钮已启用
5. **尝试直接入库（会被拒绝）：**
   - 即使通过其他方式触发入库，系统也会弹出错误提示：
   - `已退回样本不能直接入库，请先执行重新接收操作`
   - 确认后样本状态仍为"已退回"，历史记录和备注保持不变
6. **正确操作流程：**
   - 点击"← 重新接收"按钮
   - 输入备注：`资料补充完整，重新接收`
   - 确认后状态变为"已接收"（蓝色背景）
7. 现在"→ 已入库"按钮已启用，点击该按钮
8. 输入备注：`复核无误，正常入库`
9. 确认后状态变为"已入库"，历史记录完整记录整个流程

### 步骤 7：查看时间线
1. 双击任意样本行，或选中后点击"📋 查看时间线"
2. 查看样本基本信息和完整状态变更历史
3. 确认刚才的"退回→重新接收→入库"流程完整记录

### 步骤 8：编辑备注
1. 选中一条样本，点击"📝 编辑备注"
2. 填写破损或缺失管信息，保存
3. 列表中对应列会显示备注内容

### 步骤 9：按条件导出
1. 设置筛选条件：
   - 状态：选择"已入库"
   - 日期从：填写今天日期（格式：YYYY-MM-DD）
2. 点击"🔍 查询"
3. 点击"💾 导出数据"
4. 选择导出目录（建议选择桌面或其他可写目录）
5. 选择导出格式：CSV
6. 勾选"包含状态历史"
7. 点击"开始导出"
8. 查看导出成功提示，记录文件路径

### 步骤 10：测试导出失败场景
1. 再次点击"💾 导出数据"
2. 在导出目录中输入一个不存在的路径，如 `Z:\nonexistent`
3. 点击"开始导出"
4. 系统提示"导出目录不存在"
5. 验证：原导出文件未被修改或覆盖

### 步骤 11：持久化验证
1. 关闭程序
2. 重新运行 `python main.py`
3. 验证：
   - 所有样本数据和状态依然存在
   - 操作员配置已保存
   - 上次导出目录被记住
   - 状态历史完整保留
   - 破损/缺管备注完整

## 状态说明

| 状态代码 | 中文名称 | 说明 |
|-----------|----------|------|
| RECEIVED | 已接收 | 样本刚登记 |
| PENDING_INFO | 待补资料 | 需要补充资料 |
| STORED | 已入库 | 已完成入库 |
| RETURNED | 已退回 | 已退回委托方 |
| VOIDED | 已作废 | 样本作废 |

## 状态流转规则

### 完整状态流转图

```
已接收 → 待补资料 → 已入库 / 已退回 / 已作废
         ↓
      待补资料 → 已入库 / 已退回 / 已作废
         ↓
      已入库 → 已作废
         ↓
      已退回 → 重新接收 → 已接收 → ... / 已作废
         ↓
      已作废（终止状态）
```

### 各状态允许的流转（与代码 valid_transitions 一致）

| 当前状态 | 允许流转到 | 说明 |
|---------|-----------|------|
| **已接收** (RECEIVED) | 待补资料、已入库、已退回、已作废 | 刚登记的样本 |
| **待补资料** (PENDING_INFO) | 已入库、已退回、已作废 | 等待补充资料 |
| **已入库** (STORED) | 已作废 | 已完成入库 |
| **已退回** (RETURNED) | **已接收（重新接收）**、已作废 | ❌ 禁止直接到已入库 |
| **已作废** (VOIDED) | （无） | 终止状态 |

### 已退回样本入库的强制规则（与代码行为一致）

1. **GUI层防护**：选中已退回样本时，"→ 已入库"按钮自动禁用，"← 重新接收"按钮自动启用
2. **数据库层防护**：即使绕过界面直接调用服务或数据库层，也会被明确拒绝
3. **失败操作零影响**：状态、历史记录、备注完全不变，无二次确认对话框，无异常标记
4. **正确业务流程**：必须先点击"← 重新接收" → 状态变为"已接收" → 才能正常点击"→ 已入库"

> **强制约束**：已退回样本不能直接入库，必须先执行"重新接收"操作，将状态恢复为"已接收"后，才能正常流转到"已入库"。这是为了确保业务流程的可追溯性，每一次入库操作都必须对应明确的接收动作。

## 常见问题

**Q: 导入 CSV 时提示编码错误？**
A: 请确保 CSV 文件为 UTF-8 或 GBK 编码。

**Q: 数据库文件可以迁移到另一台电脑？**
A: 可以，直接复制 `sample_registry.db` 文件即可。

**Q: 如何清空所有数据？**
A: 关闭程序后删除 `sample_registry.db` 文件，重新启动程序即可。
